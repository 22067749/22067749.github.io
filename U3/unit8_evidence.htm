Evidence for Mobile Apps Development

I had to using my experience from my investigation, it’s now time to design the mobile app. Before I had to create the app, I needed to produce a detailed design of the prototype app that I needed to create for CambriaTek and then I had to justifying my design decisions.

My design needed meet the user requirements and intended purpose of the app that CambriaTek has asked for.

The design documents I needed to include
 are:

To detailed proposed solution, using a range of suitable tools (e.g. flowcharts, control structures, pseudocode, events).

To description of the main program tasks (data input and output, screen layouts and navigation).

To descriptions of the method of solution (how does each screen help the children learn the alphabet).

To list of any pre-defined programs/code snippets (including any functions or sub-routines).

To fully-referenced source table, documenting the sources appropriately.

To do a test plan, to test for the logic and functionality.
