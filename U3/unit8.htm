Mobile Apps Development

For this unit describes the objectives of a unit on mobile app development. The unit aims to teach students how to design, develop, test and review their own mobile app. The emphasis is on integrating predefined programs/code snippets with ready-made and original assets by using some original code instead of producing large amounts of original code from scratch.

Learning aims for this unit are:

A: To understand the characteristics and uses of mobile apps.
Mobile apps are computer programs that instruct a computer’s Central Processing Unit (CPU) to carry out the set of specific instructions given in a program for a specific reason and use.

B: To design a mobile app.
Software development life cycle, including:
Requirements of the problem.
Design specification (i.e. scope, inputs, outputs, processing, user interface).
Constraints (i.e. programming language and timescales for development).
Develop code.
Test.
Maintain code.

C: To develop and test a mobile app.
Preparing content to develop an app
Prepare and gather pre-defined programs, snippets and/or subroutines, and ready-
made and original assets.
 Edit (using appropriate editing software) and optimise assets for a mobile platform
(e.g. sacrificing quality for smaller file size).
Use file formats that are appropriate for the intended platform.

D: To review the finished mobile app.
Review the finished mobile app for:
User requirements.
 Fitness for purpose.
Constraints (e.g. programming language, time, copyright, device capabilities - connectivity and screen size).
